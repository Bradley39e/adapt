symbol

Description

The symbol attribute identifies the concept for symbols.

The symbol attribute accepts a numeric reference number.

A personalization agent can then load alternative symbols that the user is familiar with so they do not have to learn new symbols for different applications.

In some languages words and symbols are conjugated with sex or tense. In these case more then one value maybe need to map to a symbol. Authors should join values that map to a single conjugated word with a \ + \ sign with a space on each side of the \+\ sign. (User agent may choose to accept values for a conjugated term that have been separated with out the spaces.) See best practices for symbol values.

When there is more than one concept that are not part of a single conjugated term, multiple concepts can be referenced by separating them with white space. The order of multiple concepts should be the same as used in typical speech in the natural language of the content.

For a list of current reference numbers please refer to our data-symbol wiki page. The reference numbers are the same references numbers used in Bliss (BCI numbers). Here is a link to the copyright licensing from Bliss.

Examples

Here are some examples using the symbol attribute.

    	   <span data-symbol="13621">Cup</span> <span data-symbol="12324">of</span> <span data-symbol="17511" >Tea</span>
You could also say 
            <span data-symbol="13621 12324 17511">cup of Tea</span>
or 
	 <img src="cuptea.png" alt="cup of Tea" data-symbol="13621 12324 17511" alt="cup of Tea"/>

When there is more than one concept that are not part of a single conjugated term, multiple concepts are referenced by separating them with white space.

	 <img src="her-name.png" alt="שמה" data-symbol="15691 + 14707"/>

In this example a symbol is used for "her name" for the conjugated Hebrew word, שמה. The / + / is used to join the conjugated values, "her" (14707) and "name" (15691). If the sex is not important you can just use the value for name (15691).
